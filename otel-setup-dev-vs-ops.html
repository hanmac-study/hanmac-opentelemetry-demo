OpenTelemetry: Dev vs Ops 접근 방식

👨‍💻

개발자 (Dev)

애플리케이션 코드에 직접 계측을 추가하는 개발자 중심 접근

  • 코드 레벨 계측: 소스 코드에 직접 추적 코드 삽입
  • 라이브러리/SDK 사용: 언어별 OpenTelemetry SDK 활용
  • 세밀한 제어: 특정 함수, API 엔드포인트 추적
  • 커스텀 메트릭: 비즈니스 로직에 맞는 메트릭 정의
  • 개발 단계 통합: CI/CD 파이프라인에 포함
⚙️

운영팀 (Ops)

인프라 레벨에서 자동 계측을 구성하는 운영 중심 접근

  • 자동 계측: 코드 변경 없이 관측성 추가
  • 에이전트/사이드카: 별도 프로세스로 데이터 수집
  • 인프라 모니터링: 시스템 메트릭, 로그 수집
  • 중앙 집중식 관리: 여러 서비스 통합 관리
  • 운영 단계 배포: 기존 시스템에 비침투적 적용

주요 차이점

접근 시점

Dev: 개발 단계에서 계획적으로 추가

접근 시점

Ops: 운영 중인 시스템에 사후 적용 가능

구현 방식

Dev: SDK/라이브러리를 통한 명시적 계측

구현 방식

Ops: 자동 계측 에이전트, 프록시 활용

장점

Dev: 정밀한 추적, 비즈니스 컨텍스트 포함

장점

Ops: 빠른 적용, 코드 변경 불필요

적합한 상황

Dev: 새 프로젝트, 특정 기능 상세 모니터링

적합한 상황

Ops: 레거시 시스템, 전체 인프라 모니터링
